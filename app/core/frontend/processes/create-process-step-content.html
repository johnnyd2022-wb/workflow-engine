What are process steps?

Process steps allow you to define step-by-step workflows that automate your inventory management and provide complete source-to-sale traceability. Each process step represents a physical operation in your manufacturing workflow.

Use the name you already call this process internally. Example: Mixing, Fermentation, Tablet Pressing.

Define Inputs

Inputs are the physical materials you add to this step. Add from items in stock below, or define other materials. Added inputs appear here (expand to edit).

Inputs added (expand to edit):

Expand an item, enter quantity and unit (and execution type when shown), then click Click to add. The input is added at the top of the page — expand it there to edit.

Select an output from a previous step below; then set quantity and unit in the form that appears above.

Define other materials used during this step that are not in your inventory (e.g. water, freshly foraged food). Set default quantity and unit or prompt at execution.

Define Outputs

Outputs are the products or materials produced by this step. These can be used as inputs in subsequent steps, creating a complete workflow chain.

Example Workflow:

Step 1 "Mixing": Outputs "Mixed Base" → Step 2 "Fermentation": Uses "Mixed Base" as input, outputs "Fermented Product" → Step 3 "Packaging": Uses "Fermented Product" as input, outputs "Final Product"

Execution Prompts

Execution prompts are custom fields recorded when this step runs. Use the Batch number field below for traceability (recommended), and add other prompts for quality control, measurements, or execution-specific information.
