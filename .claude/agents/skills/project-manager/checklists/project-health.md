# Project Health Checklist (Project Manager)

Run at each project review. A project is "healthy" only if all boxes hold.

## Structure
- [ ] Objective and business value are stated in one sentence each.
- [ ] 3–7 milestones, each with a due date and a "done" test.
- [ ] Dependencies / critical path identified.
- [ ] Timeline fits the founder's real capacity (not a full-time fantasy).

## Momentum
- [ ] Next actions are current, each with owner + date.
- [ ] Nothing has silently slipped since last review (compare statuses).
- [ ] Top 3 risks have concrete mitigations, not just worries.
- [ ] Blockers are surfaced (not waiting for the weekly update).

## Boundaries & handoffs
- [ ] Still genuinely a *project* (change), not repeatable operations → else recommend Biz-E.
- [ ] Customer-visible value? → marketing brief to Marketing Director + note to Sales Manager.

## Report
- [ ] Done / Blocked / Next written for this review.
- [ ] `projects/<business>/ACTIVE_PROJECTS.md` updated.
- [ ] Priority conflicts (if any) escalated to Business Operator.
