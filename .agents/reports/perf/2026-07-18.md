## Worked the checklist top to bottom

1. **`app/core/backend/backend.py`** (priority 25 → cleared) — one real N+1
   (`list_processes`: `process.steps` lazy-loaded per process, now batch-fetched);
   three false positives from static analysis not seeing cross-function eager-loading,
   justified with `# nosemgrep`; two per-row `FOR UPDATE` locking loops
   (`complete_step`, `record_wastage`) left as-is and justified — batching would change
   lock-acquisition order and continue-on-error semantics in inventory-quantity code,
   not a change to make without explicit sign-off.
2. **`app/core/backend/reconciliation_service.py`** (priority 17 → cleared) — one real
   N+1 (`_enrich_producing_step_name` fetching process-with-steps per process_id in a
   loop; added `ProcessRepository.get_processes_with_steps` as the batch counterpart);
   one bounded-by-execution_id query justified; one pre-existing `# nosemgrep` comment
   that was on the wrong line (closing paren instead of match-start) and so never
   actually suppressed — moved it.
3. **`app/core/backend/checks/untracked_items.py`** (priority 14 → cleared) — same
   per-process_id N+1 as #2, same batch-method fix; the exception-fallback direct query
   justified (a LIMIT there would silently drop items needing reconciliation, not bound
   a hot path).
4. **`app/core/frontend/js/common.js`** (priority 13 → cleared) — semgrep rule bug:
   `setinterval-result-discarded` used `pattern-not` (exact-range match) instead of
   `pattern-not-inside` (containment) to exempt `const x = setInterval(...)`, so it
   never actually excluded the assigned form. Fixed the rule; `animateProgress()` was
   already storing and clearing its interval correctly.
5. **`app/core/utils/resetdb.py`** (priority 13 → cleared) — per-item repo lookup
   justified: dev/demo-only seed script over a small fixed list, not production
   traffic.
6. **`app/core/backend/checks/expired_materials.py`**, **`output_ready_date_check.py`**,
   **`app/core/backend/dagtraversal.py`** (priority 11 each → cleared) — all three are
   correctness-critical unbounded queries (a recall/compliance check, a DB-side-filtered
   ready-date check, a DAG traversal needing the complete graph) where a `.limit()` would
   silently drop qualifying rows rather than bound a hot path. Justified with
   `# nosemgrep` + reason in each.
7. **`app/core/frontend/js/core-api.js`**, **`execution-modal.js`**,
   **`app/ui/shared/password-policy.js`** (priority 11 each → cleared) — semgrep rule
   gap: `sequential-independent-awaits` excluded `$B($A, ...)` and `$B($A.$FIELD, ...)`
   but not `$A.$METHOD(...)` (e.g. `const data = await response.json()` after
   `const response = await fetch(...)`) — exactly as dependent, different AST shape.
   Added the missing exclusion.

Final: `perf_triage.py --check` → 0 static findings, 0 breaches, exit 0.
